# QA Report — 2026-04-30

## Score: 9.0 / 10

---

## Summary

Today's cycle addressed seven issues ranging from a **silent functional regression** caused
by CSP blocking inline JavaScript (the highest-impact find this cycle) through to cosmetic
CSS omissions. No runtime blockers were present at start-of-cycle. All fixes were verified
by restarting the app and inspecting the changed files.

---

## Issues Found & Fixed

### Security — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 1 | HIGH | **CSP blocks inline `onsubmit` handlers** — `script-src 'self' cdn.jsdelivr.net` without `'unsafe-inline'` silently drops `onsubmit="return confirm(...)"` on all delete forms. Users could delete dashboards, reports, data sources, and widgets without the confirmation dialog ever appearing. | Replaced all `onsubmit` attributes with `data-confirm="…"` on every affected form (5 templates). Added a CSP-safe event listener in `main.js` that intercepts `form[data-confirm]` on submit. |
| 2 | HIGH | **CSP blocks inline `<script>` in `report_view.html`** — the inline `<script>` block (KPI loading + chart init) was blocked by CSP, so KPI cards showed `—` and both charts on the Report view were blank. | Renamed KPI element IDs to match what `loadKPIs()` in `charts.js` already targets (`kpi-revenue`, `kpi-sales`, `kpi-customers`, `kpi-avg-order`). Added `data-chart-api` / `data-chart-type` attributes to both canvas elements so the existing auto-detect loop in `charts.js` picks them up. Removed the inline `<script>` block entirely. |
| 3 | HIGH | **Connection strings stored plaintext** (carried from 2026-04-29) | Implemented `encrypt_connection_string` / `decrypt_connection_string` helpers using `cryptography.fernet.Fernet`. Encryption is keyed from a `CONNECTION_KEY` env var and degrades gracefully to plaintext when the key is absent or the library is unavailable (broken system package handled via `except BaseException`). `new_data_source` now encrypts on write. |

### Performance / Reliability — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 4 | MEDIUM | **CSV export loaded full sales table into memory** — `api_export_sales` built a `StringIO` buffer with all rows before returning, risking OOM on large datasets. | Replaced with a streaming generator using `yield_per(500)`. Each chunk is written to a reused `StringIO` buffer and yielded immediately, keeping memory usage constant regardless of table size. |

### UX — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 5 | MEDIUM | **Register form blanked on validation error** — all fields were cleared when the server rejected the submission (duplicate username, weak password, etc.), forcing users to retype everything. | `register()` now passes `username` and `email` back to the template on error. Both input fields gain `value="{{ username or '' }}"` / `value="{{ email or '' }}"` in `register.html`. |
| 6 | MEDIUM | **Pagination rendered every page number** — `for p in range(1, total_pages+1)` could emit hundreds of `<a>` links for large datasets. This bloats the HTML and creates a minor denial-of-service surface. | Added `_page_range(current, total, window=2)` helper in `app.py`, registered as `page_range` Jinja2 global. Returns page numbers with `None` as ellipsis sentinel; always shows page 1, last page, and ±2 around the current page. All three list templates (dashboards, data sources, reports) updated to use it. |

### CSS — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 7 | LOW | **`.kpi-body` class undefined** — used in `index.html` and `report_view.html` KPI cards as a flex child, but not declared in `main.css`. Without it the KPI text block didn't grow to fill available space. | Added `.kpi-body { flex: 1; min-width: 0; }` to `main.css`. |

## Remaining TODOs

### High Priority

- **Implement `CONNECTION_KEY` rotation docs / migration path** — once `CONNECTION_KEY` is
  set and data is encrypted, re-keying requires a migration script. Add a
  `scripts/rotate_connection_key.py` utility.
- **Audit log table** — add `AuditLog(user_id, action, resource_type, resource_id, ip, ts)`
  and write an entry on every create/update/delete action.

### Medium Priority

- **DB-agnostic date truncation** — `func.strftime('%Y-%m', Sale.sale_date)` in
  `api_sales_overview` is SQLite-specific. Wrap in a helper for PostgreSQL compatibility.
- **Widget JSON schema validation on read** — `Widget.config` is server-generated from a
  whitelist but should be validated on read to harden against future code paths.
- **Host Chart.js locally** — eliminates CDN network dependency (SRI already mitigates
  supply-chain risk but local hosting removes the outbound request).

### Low Priority

- **Admin RBAC** — `User.is_admin` exists but no route checks it. Add `admin_required`
  decorator and an `/admin` management view.
- **Unit / integration tests** — zero coverage. Add `pytest` fixtures for route smoke
  tests and model methods.
- **Pagination count query caching** — each paginated page runs a `COUNT(*)` query; add
  caching for large tables.
