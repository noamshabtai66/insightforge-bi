## Issues Found & Fixed

### 1. `datetime.utcnow()` Deprecation ‚Äî FIXED
**Severity:** Medium  
**Files:** `app.py`, `models/models.py`, `seeds/seed.py`

`datetime.utcnow()` is deprecated since Python 3.12 and emits `DeprecationWarning` at
runtime. All call sites replaced with `datetime.now(timezone.utc)`. A private helper
`_utcnow()` added to `models.py` for use as SQLAlchemy column defaults (callable form
required).

### 2. `RevenueTarget.year` Hardcoded to `2024` ‚Äî FIXED
**Severity:** Medium (data correctness)  
**File:** `models/models.py:155`

The column default was `default=2024`, now two years stale. Changed to:
```python
default=lambda: datetime.now(timezone.utc).year
```
New revenue target rows now default to the current calendar year automatically.

### 3. Missing `Content-Security-Policy` Header ‚Äî FIXED
**Severity:** High (security)  
**File:** `app.py`

No CSP header was sent, allowing inline scripts and unrestricted source loading. Added
a strict policy via `add_security_headers()`:
```
default-src 'self';
script-src 'self' https://cdn.jsdelivr.net;
style-src 'self' 'unsafe-inline';
img-src 'self' data:;
font-src 'self';
connect-src 'self'
```
The CDN allowlist covers the Chart.js 4 bundle loaded from `cdn.jsdelivr.net`.

### 4. No Startup Warning for Weak `SECRET_KEY` ‚Äî FIXED
**Severity:** Medium (security ops)  
**File:** `app.py`

The fallback `SECRET_KEY = 'dev-secret-change-in-production'` was silent. Added a
`sys.stderr` warning at startup if the default key is detected, making misconfigured
production deployments immediately obvious in logs.

### 5. `data_type` Field Missing Server-Side Allowlist ‚Äî FIXED
**Severity:** Medium (security + data integrity)  
**File:** `app.py` ‚Äî `new_data_source` route

The form offered `csv`, `database`, `api`, `excel` but the backend accepted any string.
Added `ALLOWED_TYPES = {'csv', 'database', 'api', 'excel'}` with explicit 400-path
flash if an unexpected value is posted.

### 6. No Max-Length Input Validation on Name Fields ‚Äî FIXED
**Severity:** Low (robustness)  
**File:** `app.py`

Dashboard, report, and data-source name fields lacked server-side length checks,
allowing strings up to the DB column limit (120 chars) to pass silently or‚Äîwith a
future DB change‚Äîcause truncation errors. All three POST handlers now reject names
longer than 120 characters with a clear flash message.

### 7. `Sale.total_amount` Incorrectly Nullable ‚Äî FIXED
**Severity:** Low (schema correctness)  
**File:** `models/models.py:131`

`total_amount` was `nullable=True` but is always computed as `price √ó quantity` in the
seeder and all future write paths. Changed to `nullable=False` to enforce the invariant
at the DB level.

### 8. `index.html` ‚Äî Recent Reports Not Clickable ‚Äî FIXED
**Severity:** Low (UX inconsistency)  
**File:** `templates/index.html`

Report list items were plain `<span>` text. Dashboard items used `<a href>` links.
Fixed reports to link to `view_report` for consistency.

### 9. `index.html` ‚Äî `top_products` Data Passed but Not Displayed ‚Äî FIXED
**Severity:** Low (dead code / wasted query)  
**File:** `templates/index.html`

The `index` route computed and passed `top_products` (5-product revenue query) but the
template had no corresponding section. Added a "Top 5 Products by Revenue" list card
below the Recent Dashboards / Recent Reports section, rendering the data with formatted
currency.

### 10. `python-dotenv` Not in Dependencies ‚Äî FIXED
**Severity:** Low (developer experience)  
**File:** `requirements.txt`

`python-dotenv` was a noted TODO. Added `python-dotenv==1.0.1` to `requirements.txt`
and added a graceful `try/except ImportError` import in `app.py` that calls
`load_dotenv()` so `.env` files are loaded automatically when the package is present.
