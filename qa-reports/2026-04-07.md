## Security Improvements

### 3. Rate limiting on `/login` and `/register` (Flask-Limiter)
- **Files:** `requirements.txt`, `app.py`
- **Problem:** Brute-force attacks on the authentication endpoints were unbounded — a carry-forward HIGH priority item from QA 2026-04-05.
- **Fix:**
  - Added `Flask-Limiter>=3.5,<4` to `requirements.txt`.
  - Initialised `Limiter(get_remote_address, storage_uri='memory://')` in `app.py`.
  - Applied `@limiter.limit('20 per minute; 100 per hour', methods=['POST'])` to `/login`.
  - Applied `@limiter.limit('10 per minute; 30 per hour', methods=['POST'])` to `/register`.
  - Added HTTP 429 error handler so rate-limited responses render the styled error page.
- **Note:** In-memory storage is appropriate for a single-process deployment. For multi-process/multi-server setups, switch `storage_uri` to `redis://…`.

### 4. CSP `style-src 'unsafe-inline'` removed
- **File:** `app.py:add_security_headers()`
- **Problem:** The Content-Security-Policy previously allowed inline styles (`'unsafe-inline'`), which weakens XSS protection.
- **Fix:** All inline `style="…"` attributes were removed from every template (six files) and replaced with proper CSS classes (see Fix #5). The CSP is now `style-src 'self'` with no `'unsafe-inline'` exemption.
