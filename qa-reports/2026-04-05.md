## Security Issues Fixed

### 2. Hardcoded trivial default admin password (`admin123`)
- **File:** `app.py:create_tables()`
- **Problem:** The auto-created admin account used a well-known password. Any unattended deployment was trivially compromised.
- **Fix:** `create_tables()` now reads `ADMIN_PASSWORD` from the environment. If unset, a cryptographically random password (`secrets.token_urlsafe(14)`) is generated and logged as a one-time warning. The old literal `'admin123'` is gone.

### 3. XSS-unsafe delete confirm dialogs (reports + dashboard list)
- **Files:** `templates/report_view.html`, `templates/dashboards.html`
- **Problem:** `confirm('Delete this report?')` embedded no name, and any future attempt to include the name directly in the string would be vulnerable to XSS via crafted dashboard/report names.
- **Fix:** Both forms now use the `data-name` attribute pattern already in use on `dashboard_view.html`:  
  `onsubmit="return confirm('Delete report &quot;' + this.dataset.name + '&quot;?')"`  
  The name is read from a DOM attribute (auto-escaped by Jinja2), never concatenated into the JS string.

## Code Quality Fixes

### 4. `Float` columns for monetary values → `Numeric(precision, scale)`
- **File:** `models/models.py`
- **Problem:** `Product.price`, `Sale.total_amount`, `Employee.salary`, and `RevenueTarget.target_amount` used `db.Float`, which is subject to IEEE-754 rounding errors — unacceptable for financial data in a BI platform.
- **Fix:** Migrated to `db.Numeric` with explicit precision/scale:
  - `Product.price` → `Numeric(10, 2)`
  - `Sale.total_amount` → `Numeric(10, 2)`
  - `Employee.salary` → `Numeric(12, 2)`
  - `RevenueTarget.target_amount` → `Numeric(14, 2)`
- **Note:** `db.create_all()` does not ALTER existing tables. A fresh database (delete `insightforge.db` and re-run `python app.py`) picks up the new types. A migration tool (Alembic) is needed for live data — see TODOs.

### 5. JSON serialisation of `Decimal` values
- **File:** `app.py` (all API endpoints)
- **Problem:** With `db.Numeric`, SQLAlchemy returns `decimal.Decimal` objects which Flask's `jsonify` cannot serialise natively.
- **Fix:** All numeric API values are wrapped in `float()` before `round()` to ensure clean JSON output.

## UI/UX Fixes

### 6. Auth page double-padding layout bug
- **File:** `static/css/main.css`
- **Problem:** `.auth-page` used `min-height: 100vh` to vertically centre the sign-in card, but it is rendered inside `.main-content` which has `padding: 28px 24px`. This pushed the card ~28px below true centre and caused scroll on short viewports.
- **Fix:** Added `margin: -28px -24px` to `.auth-page` to escape the parent padding, restoring true full-viewport centering.
