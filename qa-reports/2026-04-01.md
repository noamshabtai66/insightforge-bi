## Critical Issues Found & Fixed

### 1. Seed data date range — all charts were empty (CRITICAL)
- **File:** `seeds/seed.py`
- **Issue:** Sales were seeded with `start_date = datetime(2021, 1, 1)` + max 730 days,
  placing all data between 2021-01 and 2023-01. The overview chart shows the **last 12 months**
  (2025-04 – 2026-04). No seed data fell inside this window → all charts rendered blank,
  all KPIs showed zero.
- **Fix:** Changed to dynamic range: `end_date = datetime.utcnow()`,
  `start_date = end_date - timedelta(days=730)`. Also fixed `RevenueTarget.year`
  from hardcoded `2024` to `datetime.utcnow().year`.
- **Verified:** After fix `/api/kpis` returns `total_revenue: $4,120,520`, `total_sales: 5000`;
  `/api/sales-overview` returns 13 months of data up to 2026-04.

### 2. `debug=True` hardcoded in `app.run()` (SECURITY/CRITICAL)
- **File:** `app.py:419`
- **Issue:** Flask debug mode was unconditionally enabled, exposing the interactive debugger
  to any network client on port 5000 — allows arbitrary Python execution.
- **Fix:** `debug = os.environ.get('FLASK_DEBUG', 'false').lower() == 'true'`

### 3. Stale session crash — deleted user causes AttributeError (BUG)
- **File:** `app.py` — `login_required` decorator
- **Issue:** `login_required` only checked `'user_id' in session`, not whether the user
  still exists. If the user record was deleted, `current_user()` returned `None` and
  every route calling `user.id` would raise `AttributeError: 'NoneType' has no attribute 'id'`.
- **Fix:** Added a DB lookup inside `login_required`; if user is gone, clears the session
  and redirects to login with an informative flash message.

### 4. `base.html` nav active check — TypeError on 404/500 pages (BUG)
- **File:** `templates/base.html:17-19`
- **Issue:** `'dashboard' in request.endpoint` raises `TypeError: argument of type 'NoneType'
  is not iterable` when `request.endpoint` is `None` (which happens on unhandled 404/500).
- **Fix:** Added `request.endpoint and` guard before each `in` check.

## Security Issues Found & Fixed

| Issue | Severity | Fix Applied |
|-------|----------|-------------|
| `debug=True` hardcoded — interactive debugger exposed | Critical | Reads `FLASK_DEBUG` env var; defaults to `false` |
| No HTTP security headers | Medium | Added `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`, `Referrer-Policy` via `@after_request` |
| Session cookies lacked `HttpOnly` / `SameSite` flags | Medium | Added `SESSION_COOKIE_HTTPONLY = True` and `SESSION_COOKIE_SAMESITE = 'Lax'` to app config |
| Full connection strings displayed in data sources table (potential password exposure) | Low | Truncated to first 20 chars + `…` in `data_sources.html` |
