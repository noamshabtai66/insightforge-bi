Today's cycle focused on the three highest-priority items carried over from the 2026-04-20

## Issues Found & Fixed

### Security — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 1 | HIGH | No account lockout — IP-rate-limit could be bypassed by rotating IPs, leaving passwords open to distributed brute force | Added `failed_logins` (int) and `locked_until` (DateTime) to `User` model. After 5 consecutive failures the account is locked for 15 minutes. Lock is cleared automatically on expiry or on a successful login. |
| 2 | MEDIUM | App silently started with the insecure default `SECRET_KEY` even when `FLASK_ENV=production`, risking session forgery | Added hard `RuntimeError` at startup when `FLASK_ENV=production` and the default key is detected; prints a generation command in the error message. |
| 3 | LOW | `ensure_columns()` migration function absent — adding new DB columns would break existing deployments silently | Added `ensure_columns()` that runs on every startup, uses SQLAlchemy `inspect` to detect missing columns, and issues `ALTER TABLE` DDL to add them. Called before `db.create_all()` in `create_tables()`. |

### Code Quality — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 4 | LOW | Unused import: `from flask_sqlalchemy import SQLAlchemy` in `app.py` — `SQLAlchemy` is never referenced there | Removed the dead import. |
| 5 | LOW | `view_dashboard` route passed two unused template variables (`widget_data_sources`, `allowed_widget_types`) that bloat context without being rendered | Removed both from the `render_template` call. |
| 6 | LOW | `Widget.position` column had no DB index — `ORDER BY Widget.position` on dashboards with many widgets incurred a full table scan | Added `index=True` to `Widget.position` in the model. |
| 7 | LOW | `models/` directory had no `__init__.py` — relied on implicit namespace packages (PEP 420), which behaves differently across tools | Created empty `models/__init__.py` to make it an explicit package. |

### UX — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 8 | LOW | Description textareas in `dashboard_form.html` and `report_form.html` had no `maxlength` attribute — server validates 1000 chars but browser gave no client-side feedback | Added `maxlength="1000"` to both textareas, matching server-side constraint. |
| 9 | LOW | Connection string input in `data_source_form.html` had no `maxlength` — could allow very long strings to reach the server | Added `maxlength="2000"` (generous for DSNs, prevents trivial oversized input). |
| 10 | LOW | `.card-grid` (Dashboards and Reports list pages) had no `margin-bottom`, causing items to run flush into the pagination nav | Added `margin-bottom: 24px` to `.card-grid` in `main.css`. |

## What Was Already Good

- **CSRF protection** — constant-time token comparison, injected into every POST form
- **Rate limiting** — Flask-Limiter on all auth and API routes
- **Security headers** — CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, HSTS (HTTPS only)
- **Open redirect prevention** — `_safe_next()` validates next-URL targets
- **Ownership checks** — all user resources filtered by `user_id`
- **Password hashing** — Werkzeug PBKDF2-SHA256 via `generate_password_hash`
- **Session cookie flags** — HTTPOnly, SameSite=Lax, optional Secure
- **Input validation** — length + type allowlists on all form fields, widget types, data source types
- **Pagination bounds** — `MAX_PAGE=10000` prevents DoS via absurd page numbers
- **MAX_CONTENT_LENGTH** — 2 MB limit blocks oversized POST payloads
- **Session fixation mitigation** — `session.clear()` before writing new session on login
- **CSV export** — authenticated, rate-limited, streams correct Content-Disposition header
- **Health check** — `/api/health` liveness probe without auth, reports DB status

## Remaining TODOs (Not Fixed This Cycle)

### High Priority

- **Encrypt connection strings at rest** — `DataSource.connection_string` is stored as
  plaintext. Implement `Fernet` symmetric encryption keyed from a `CONNECTION_KEY` env var,
  encrypting on write and decrypting on read.
- **Audit log table** — add `AuditLog(user_id, action, resource_type, resource_id, ip, ts)`
  and write an entry on every create/update/delete action. Required for compliance.

### Medium Priority

- **DB-agnostic date truncation** — `func.strftime('%Y-%m', Sale.sale_date)` in
  `api_sales_overview` is SQLite-specific. Wrap in a helper that emits
  `to_char(... 'YYYY-MM')` for PostgreSQL.
- **Streaming CSV export** — current `/api/export/sales` loads all rows into a `StringIO`
  buffer before streaming. Replace with a generator using `yield_per(500)` for large tables.
- **Widget JSON schema validation** — `Widget.config` is server-generated from a whitelist
  (`WIDGET_DATA_SOURCES`), so the risk is low, but add explicit schema validation on read
  to harden against future code paths that might bypass the allowlist.

### Low Priority

- **Admin RBAC** — `User.is_admin` exists but no route checks it. Add an
  `admin_required` decorator and protect an `/admin` dashboard.
- **Unit/integration tests** — zero coverage. Add `pytest` fixtures for route smoke tests
  and model methods (at minimum `set_password` / `check_password` and the lockout logic).
- **Host Chart.js locally** — reduces CDN dependency (SRI hash already mitigates the
  main supply-chain risk, but local hosting eliminates the network dependency entirely).
- **Pagination performance** — count queries run on every paginated page; add a
  `func.count()` index or cache the total for large tables.
