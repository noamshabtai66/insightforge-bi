# QA Report — 2026-04-20

## Score: 7.5 / 10

---

## Summary

InsightForge BI is a well-structured Flask BI platform with solid fundamentals: CSRF
protection, rate limiting, proper password hashing, security headers, and ownership
checks on all user resources. No critical runtime blockers were found after dependency
installation. This cycle focused on hardening security, closing several small attack
surface areas, and shipping visible feature improvements.

## Issues Found & Fixed

### Security — Fixed

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 1 | HIGH | No `MAX_CONTENT_LENGTH` — POST requests could be unbounded (DoS) | Added `MAX_CONTENT_LENGTH = 2 MB` to Flask config + 413 error handler |
| 2 | HIGH | Session fixation: CSRF token / session not cleared on login | `session.clear()` before writing new session on successful login |
| 3 | MEDIUM | Username allowed arbitrary characters (injection surface, confusion attacks) | Added `_USERNAME_RE = /^[a-zA-Z0-9_-]+$/` regex validation in register route |
| 4 | MEDIUM | `page` query param unbounded — DoS via `?page=999999999` | Capped all paginated routes to `min(page, MAX_PAGE=10000)` |
| 5 | LOW | No logging of failed login attempts (impedes intrusion detection) | Added `logger.warning` on failed auth + `logger.info` on success |

## What Was Already Good

- **CSRF protection** — constant-time token comparison with `secrets.compare_digest`
- **Rate limiting** — Flask-Limiter on all auth and API routes
- **Security headers** — CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, HSTS (HTTPS only)
- **Open redirect prevention** — `_safe_next()` validates redirect targets
- **Ownership checks** — all user-owned resources filtered by `user_id`
- **Password hashing** — Werkzeug `generate_password_hash` / `check_password_hash`
- **Session cookie flags** — HTTPOnly, SameSite=Lax, optional Secure
- **Input validation** — length checks on all form fields, type allowlists on data sources and widgets
- **Responsive UI** — mobile hamburger nav, responsive grids

## Remaining TODOs (Not Fixed This Cycle)

### High Priority

- **Encrypt connection strings at rest** — currently stored as plaintext `Text` column; consider AES-256 with a KMS-managed key or at minimum `Fernet` symmetric encryption before storing.
- **Account lockout** — no per-user failed-login counter; brute force with rotating IPs bypasses the IP-based rate limit. Add a `failed_logins` + `locked_until` field to `User`.
- **`SECRET_KEY` hard-fail in production** — app starts with the insecure default. Add a startup check that raises `RuntimeError` when `FLASK_ENV=production` and the default key is detected.

### Medium Priority

- **DB-agnostic queries** — `func.strftime('%Y-%m', ...)` is SQLite-only; wrap in a helper that emits `to_char(... 'YYYY-MM')` for PostgreSQL.
- **Audit log table** — add an `AuditLog` model recording `(user_id, action, resource_type, resource_id, ip, timestamp)` for compliance.
- **Widget JSON schema validation** — widget `config` column is unvalidated JSON; a bad `api_endpoint` value stored there could result in the frontend fetching unexpected URLs.
- **Pagination on CSV export** — the current export loads all rows into memory; add a `LIMIT`/`OFFSET` streaming approach or use SQLAlchemy `yield_per` for large datasets.

### Low Priority

- **Unit/integration tests** — zero test coverage; add pytest fixtures for models and route smoke tests.
- **Host Chart.js locally** — reduces CDN dependency risk; SRI hash is already present which mitigates the main supply-chain risk.
- **Admin RBAC** — `is_admin` field exists but is never checked in any route decorator; either wire it up or remove it.
- **`func.max` on widgets without index** — `Widget.position` has no database index; add one for dashboards with many widgets.
